Cucumber JVM follows [Semantic Versioning](https://semver.org/spec/v2.0.0.html). The public API is defined as any public class in the `cucumber.api` package or annotated with `@API(status = API.Status.STABLE)`. Breaking changes to the public API are only made in major releases. The full changelog is maintained in [CHANGELOG.md](https://github.com/cucumber/cucumber-jvm/blob/main/CHANGELOG.md) at the root of the repository, which follows the [Keep a Changelog](https://keepachangelog.com/en/1.0.0/) format. This page summarises the most recent and notable releases; follow the GitHub links for the complete list of changes in each version.

<Note>
For migration instructions between major versions, see the [Upgrading guide](/reference/upgrading).
</Note>

## Semver policy

Cucumber JVM's public API is defined as:

- Any public class in the `cucumber.api` package or a subpackage thereof.
- Any public class annotated with `@API(status = API.Status.STABLE)`.

Changes to other classes — even public ones — are not considered breaking changes under this policy. If you depend on internal classes, you may need to adapt when upgrading patch or minor versions.

For the complete history of every version, including versions prior to v7, see:
